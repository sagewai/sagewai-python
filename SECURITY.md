# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in Sagewai, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Email: security@sagewai.ai

Include in your report:

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Potential impact
- Any suggested fix (optional)

### Response Timeline

- **Acknowledgment**: Within 48 hours of receipt
- **Initial assessment**: Within 5 business days
- **Fix timeline**: Critical issues within 30 days, others within 90 days
- **Disclosure**: Coordinated disclosure after fix is released

### Scope

The following are in scope:

- Sagewai SDK (`sagewai` package)
- Admin dashboard (backend and frontend)
- CLI tools
- VS Code extension
- Official documentation site

### Out of Scope

- Third-party dependencies (report to the relevant project)
- Issues in example/demo code that are not exploitable in production
- Social engineering attacks

### Recognition

We appreciate security researchers who help keep Sagewai safe. With your permission, we will acknowledge your contribution in the release notes.
